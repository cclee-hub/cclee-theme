# UI/UX 设计规范

<!-- 开发时必须遵守的硬性规则，不包含原因解释 -->

## 颜色

### 对比度 (WCAG AA)

- 正常文本：≥4.5:1
- 大文本（≥18px 粗体 或 ≥24px）：≥3:1

### 项目颜色使用

- **accent (#f59e0b)**：白色背景对比度仅 2.9:1，禁止用于正常文本，仅限图标或大文本
- **浅色背景链接**：用 primary (#0f172a) + 下划线，不依赖颜色区分
- **新增颜色**：必须用 [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) 验证

## 交互

- 点击区域最小 44×44px
- 焦点样式：2px solid primary，outline-offset 2px
- 表单 input 必须关联 label（可用 aria-label）
- 禁用状态：opacity 0.5 + cursor not-allowed
- 过渡动画：150~400ms，ease-out

## 间距

- 基础单位：8px
- 常用值：4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px
- 移动端边距 ≥16px

## 宽度

| 用途 | 宽度 | theme.json |
|------|------|------------|
| 正文内容 | 800px | contentSize |
| 图片/卡片 | 1200px | wideSize |
| Hero/CTA | 100% | 全宽 |

禁止正文全宽（可读性差）

## 断点

| 名称 | 宽度 |
|------|------|
| sm | 640px |
| md | 768px |
| lg | 1024px |
| xl | 1280px |

## 排版

- 行高：正文 1.5~1.6，标题 1.1~1.2
- 每行字符：英文 45~75，中文 20~40

## 装饰

- 圆角：0 / 4px / 8px / 16px
- 阴影：sm / md / lg（参考 Tailwind 默认值）

# 高级感设计原则

## 呼吸感（留白）

- 宁可多，不可少
- 元素之间 ≥16px，区块之间 ≥48px
- 页面边缘 ≥24px（移动端 ≥16px）
- 紧凑 = 廉价感，宽松 = 高级感

## 视觉层次

- 标题 vs 正文：字号差距 ≥1.5 倍
- 主操作 vs 次操作：颜色/大小明显区分
- 重要信息用颜色/粗体，次要信息用 neutral 色

## 氛围营造

- 避免纯白背景，用 neutral-50 或微妙渐变
- 阴影要柔和：大扩散 + 低透明度
- 卡片悬停：微妙上移 + 阴影加深（不是边框高亮）

## 字体

- 避开 Inter / Roboto / Arial（AI 感太重）
- 标题：选有个性字体（如 Outfit / Space Grotesk / Noto Sans SC）
- 正文：可读性优先，中文字用思源黑体 / 苹方

## 微交互

- 按钮 hover：背景加深 10% 或上移 2px
- 卡片 hover：translateY(-4px) + shadow-lg
- 输入 focus：边框变色 + 微妙外发光
- 所有动画 ease-out，避免 linear

## 一致性

- 同类元素：相同圆角、相同阴影、相同间距
- 按钮：全站统一 2~3 种样式（主/次/幽灵）
- 图标：统一线条粗细和视觉重量

# 检查清单

开发完成后逐项确认：

- [ ] accent 未用于正常文本
- [ ] 链接在浅色背景有下划线或用 primary 色
- [ ] 可点击元素 ≥44×44px
- [ ] 焦点状态可见（键盘导航测试）
- [ ] 表单 input 有关联 label
- [ ] 禁用状态有视觉提示
- [ ] 间距为 8 的倍数
- [ ] 正文宽度 ≤800px
- [ ] 区块间距 ≥48px（呼吸感）
- [ ] 标题字号 ≥1.5 倍正文（视觉层次）
- [ ] 背景非纯白（氛围）
- [ ] 同类元素样式一致
- [ ] Skip to content 链接存在（可用 CSS 隐藏）
