检查项目代码变更，同步更新 CONTEXT.md 文档。

## 执行流程

### 步骤 1：检查代码变更

对比最近提交与 CONTEXT.md 内容，检查以下项目：

| 检查项 | 对应 CONTEXT.md 章节 |
|--------|---------------------|
| 新增/删除子目录 | 子项目 |
| 新增文档文件 | 文档 |
| 配置变更（端口、数据库等） | WordPress 本地开发环境 |
| 部署流程变更 | WordPress 部署流程 |
| 服务器路径变更 | 目录映射 |

### 步骤 2：生成同步建议

如有不一致，输出表格：

```markdown
## CONTEXT.md 同步建议

| 章节 | 变更项 | 当前文档值 | 代码实际值 |
|------|--------|-----------|-----------|
| 子项目 | 新增目录 | — | `new-module/` |
| 配置 | 端口 | 8080 | 3000 |

操作选项：
1. 确认更新
2. 跳过
```

### 步骤 3：执行更新

用户确认后，使用 Edit 工具更新 CONTEXT.md。

## 输出格式

**无需更新时：**
```
✅ CONTEXT.md 与代码一致，无需更新。
```

**有更新时：**
```
📝 已更新 CONTEXT.md：
- 子项目：新增 new-module/
- 配置：端口 8080 → 3000
```
